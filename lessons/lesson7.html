⏰ Scheduled Tasks (Automation)

Scheduled Tasks = ให้ Cowork ทำงานอัตโนมัติ แม้เรา offline

ตั้งเวลางานอัตโนมัติ

ตัวอย่างที่ใช้ได้จริง:

  • 🕐 Every Monday 9 AM: "สรุป Week reports จากเมล ส่ง Summary"
  • 🕐 Daily 5 PM: "ตรวจสอบ Patient follow-up ที่ยังไม่ตอบกลับ ส่ง reminder"
  • 🕐 Every 1st of month: "ดึง Statistics จากทั้งเดือน สร้าง Report"
  • 🕐 Every 4 hours: "ตรวจสอบ Trial patient data ที่ upload มาใหม่"

💡 5 Advanced Tips

💡 Tip 1: Chaining (สั่งงานแบบเรียงลำดับ)

คืออะไร? ให้ Agent 1 ทำเสร็จ → ส่งผลให้ Agent 2 → เป็นไปเรื่อย ๆ

ตัวอย่าง: Download patient file → Parse → Validate → Store → Notify → Done

💡 Tip 2: Parallel Processing (ทำหลาย ๆ อันพร้อม)

คืออะไร? แทนการทำ Sequential ให้ทำพร้อมกัน ประหยัดเวลา

ตัวอย่าง: 100 patient records → split to 5 agents (20 each) → process in parallel → merge results

💡 Tip 3: Error Handling & Retries

คืออะไร? ถ้า Task fail บอก Claude ว่าต้องทำไร (retry, skip, alert)

ตัวอย่าง: "If data parse fails → retry 2 times → if still fail → send alert to manager"

💡 Tip 4: Audit & Logging

คืออะไร? ให้ Cowork บันทึก "ใครทำอะไร ตอนไหน" ไว้

สำคัญสำหรับ Healthcare (compliance): "User X ran workflow Y on [date] → 50 records processed → 2 errors"

💡 Tip 5: Cost Optimization

คืออะไร? ลด API calls ลด processing time ด้วยการ Batch processing

ตัวอย่าง: จาก "100 calls × 1 req/call" → "10 batches × 10 req/batch" = ประหยัด 70-80%

🔒 Security & Compliance for Healthcare

⚠️ Healthcare-Specific ต้องระวัง:

  1. HIPAA/PDPA Compliance: ไม่ส่ง PII (Patient ID, Name) ไปที่ Third-party
  2. Data Encryption: Connectors ต้อง SSL/TLS encrypted
  3. Access Control: ให้ Least Privilege — Cowork เข้าถึง เฉพาะ Folder ที่ต้อง
  4. Audit Trail: บันทึก "WHO accessed WHAT data WHEN" เพื่อ compliance
  5. Data Retention: ลบข้อมูล Temp หลังใช้ (ไม่เก็บ redundant data)
  6. Manual Review: สำหรับการตัดสินใจด้าน Medical ต้องมี QA by human
