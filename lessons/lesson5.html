🤖 บทที่ 5 — สร้าง Custom Skills และ Plugins

พี่หมีสอนสร้าง AI Assistant เฉพาะทางของคุณ 🛠️

Custom Skills คือ 'สูตรลับ' ที่คุณสอน Claude ให้ทำงานเฉพาะทางของคุณครับ

📚 3 Building Blocks ของ Plugin

📚 Block 1: Skills

คืออะไร? Template/workflow ที่ Claude ต้องทำซ้ำ ๆ

ตัวอย่าง:
• "Draft SOAP note template"
• "Medication reconciliation checklist"
• "Patient follow-up email"

🔌 Block 2: Connectors

คืออะไร? Integration กับ Apps (Gmail, Drive, Calendar)

ตัวอย่าง:
• ส่ง Email output
• Save to Google Docs

🤖 Block 3: Sub-agents

คืออะไร? ให้ Claude แยก Task เป็น Sub-tasks

ตัวอย่าง:
• Agent 1: Parse patient data
• Agent 2: Analyze interactions
• Agent 3: Generate report

📖 What are Skills?

Skills คือ เอกสาร (Markdown file) ที่สอน Claude ให้ทำงานเฉพาะอย่างให้ดี

📋 ตัวอย่าง Skill: "Medication Counseling Leaflet"

# Medication Counseling Leaflet Skill

## Context
You are a clinical pharmacist creating patient education materials in Thailand.

## Objective
Generate a 1-page leaflet for a specific medication that is clear, accurate, and patient-friendly.

## Process
1. Take drug name as input
2. Identify: indication, mechanism, dosing, common SE, drug interactions
3. Write in simple Thai (high school level)
4. Include: "When to call doctor" section

## Output Format
- Title + drug name
- 5 sections (Why, How, Side Effects, Diet, When to call)
- Bullet points only (no long paragraphs)
- Include 1 visual separator

## Healthcare-Specific Rules
- Never exceed 1 page
- Emphasize drug-food interactions for Thai cuisine
- Use term from Thai FDA (if available)

🔌 Connectors in Plugins

Plugins สามารถเชื่อมต่อกับ Connectors เพื่อ Automate Output:

📧 Gmail Connector

Output ที่ Plugin สร้าง → ส่ง Email ให้ Patient โดยอัตโนมัติ

📁 Google Drive Connector

Save leaflet ที่สร้าง → Google Docs folder โดยอัตโนมัติ

🤖 Sub-agents in Plugins

สำหรับงาน Complex ให้ Plugin สร้าง Sub-agents:

🏥 Healthcare Example: "Clinical Trial Screening Plugin"

Agent 1: Data Parser
อ่าน Patient records → Extract relevant data (Age, BP, Lab)

Agent 2: Criteria Checker
ตรวจสอบ Inclusion/Exclusion → Category patient

Agent 3: Report Generator
สร้าง Summary report + send to coordinator

🎓 เมื่อไหร่ควรสร้าง Custom Plugin?

✅ สร้าง Plugin เมื่อ:

  • งานซ้ำ ๆ ที่เกิดขึ้น 5+ ครั้ง/สัปดาห์
  • ต้องการ Standardization (เสมอ ๆ)
  • ต้อง Integration กับ Apps (Gmail, Drive)
  • ต้อง Automation (ไม่ต้องสั่งงาน)

❌ ไม่ต้องสร้าง เมื่อ:

  • งานเก่า ๆ ไม่ซ้ำ
  • ต้องความสร้างสรรค์สูง
  • Setup ซับซ้อนเกินไป (ROI ลบ)

🛠️ 6 Steps: สร้าง Healthcare Plugin เองจากศูนย์

Step 1: Define the Problem

เลือกงานที่ต้องการ Automate (เช่น "สร้าง Patient Counseling Material")

Step 2: Write the Skill (Markdown)

สร้าง .md file ที่สอน Claude → Output ที่ต้องการ (see example above)

Step 3: Choose Connectors

เลือก Input connector (Gmail? Drive?) + Output connector (Save where?)

Step 4: Create Sub-agents (Optional)

ถ้างาน Complex ให้แบ่งเป็น Sub-tasks

Step 5: Test with Sample Data

ลองใช้ Plugin จริง ด้วย test data ก่อน

Step 6: Deploy & Monitor

Deploy ให้ Team ใช้ → Monitor ผล → Iterate

⚙️ Plugin Management & Best Practices

5 Tips สำหรับการใช้ Plugin ได้ดี:

  1. Version Control: ให้ Version ของ Skill (v1, v2) เพื่อ Track changes
  2. Documentation: เขียน README สำหรับ Team ว่า Plugin ทำอะไร
  3. Error Handling: บอก Claude ว่าต้องทำไร ถ้าข้อมูล Invalid
  4. Monitor Output: ตรวจสอบ Output ระหว่างเดิน (โดยเฉพาะ Healthcare)
  5. Iterate & Improve: ทำ Feedback loop ให้เพิ่มประสิทธิภาพ
